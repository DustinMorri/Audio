As you can see for these example parameters, you can get a pretty interesting stereo profile where only the extreme values of the mid and side are allowed to pass through.<br/>
The side values from 0.25 to 0.75, are in-general compressed due to that inflection point compression value of 0.70, but from 0.25 to 0.50 we are also compressing it even more with that zone 1 compression value of 0.90.<br/>
The zone 2 "compression" value of 0.25 actually saturates the side values from 0.50 to 0.75. This makes the overall wave shape blend back in to what would otherwise be a perfect circle.

## GNU Octave Scripts
This folder contains Octave source code that is not included in the signal package that I found that I needed to create reverse impulse responses for custom equalization. As it turns out, dividing by 0 is also a problem for complex numbers. It's just more complex.
### soft_clip.m
This script can clip off sharp peaks of input signals.
```Matlab
%{
shape - tanh, tanhmax, alg, sin, or lin
x - a vector of values 0.0 to inputRange
inputRange - a value greater than 0.0
outputRange - a value greater than 0.0
returns a vector of values 0.0 to outputRange after clipping by selected shape function
%}
function y = soft_clip(shape, x, inputRange, outputRange)
```
### create_bell.m
This script uses the sin shape of the soft_clip script to generate a more gradual bell-shaped curve on the top of very sharp peaks of audio signals.
```Matlab
%{
width - the number of numbers you would like the output vector to have
outputRange - a value greater than 0.0 relating to the maximum value (i.e. the top of the bell curve)
%}
function y = create_bell(width,outputRange)
```
### Helper C++
GNU Octave's `load` and `save` functions work in human-readable numbers which are obviously not the wav files that digital audio workstations use. This folder contains the C++ code and x86_64 Windows-tested executables to decode and encode wav files. *Note: always be weary of downloading executables like this and try to learn how to compile code yourself.* I believe there should be a better way to do this, but maybe there really is not. This code might be useful to someone outside the scope of what I'm doing with it. Most of the heavy lifting is done by the header only dr_wav code which is an incredible library.
#### ReadWavWriteFloats
Given one 32-bit floating-point stereo input wav file, write the values from channel 0 (for left) or 1 (for right) into human-readable numbers that can be consumed by GNU Octave's `load` function.<br/>
`ReadWavWriteFloats in.wav out.txt channel`
#### ReadFloatsWriteWav
Given one text file containing a frameCount number of human-readable floating-point values generated by GNU Octave's `save` function, write a 32-bit floating-point stereo output wav file where both the left and right channels have the same value.<br/>
`ReadFloatsWriteWav in.txt out.wav frameCount`
#### CombineWavsFloat
Given two 32-bit floating-point stereo input wav files (left and right for example), this executable will produce an output wav file with the left channel from the left file and the right channel from the right file.<br/>
`CombineWavs left.wav right.wav combined.wav`

### Generated Wav Files
#### el_iso226_40phon.wav
When a signal is convolved using this file, the resulting signal will be equalized to match the curves defined by the equal loudness 40 phon international 226 standard.<br/>
As you can see, the human ear is pretty sensitive in the 2,000 to 5,000 hertz range.<br/>

#### el_iso226_40phon_opposite_truncated.wav
When mixing music, you want to quiet the frequencies that the ear is sensitive to, and louden the frequencies that the ear is not sensitive to.<br/>
In other words, you want to do the opposite of applying the equal loudness curve.<br/>
Note that the high frequencies get a little bit extreme in the math, so I limited the areas where it would divide by zero (because the average older person cannot hear higher frequencies very well) to +12.0dB.

#### log and linear_sine_sweep_1_to_22050Hz_10s.wav
These files are pretty self-explanatory.<br/>
I use them to determine the impulse response of any given room or speaker setup.<br/>
Note that if you want to do that, it might be a good idea to include an index noise so that you can line up impulse and response files down to the sample.<br/>
You might also want to amp the file down so that it doesn't blow up your speakers constantly trying to reproduce true -1.0 and 1.0 values.<br/>
This is what I did with the linear_sine_sweep_1_to_22050Hz_10s_with_start_index.wav file.
